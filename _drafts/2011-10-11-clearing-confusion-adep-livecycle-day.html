--- layout: post title: 'Clearing the confusion: ADEP = LiveCycle + Day CRX' date: '2011-10-11T04:09:00.000-07:00' author: Michael Steward tags: modified_time: '2012-03-03T11:39:27.126-08:00' blogger_id: tag:blogger.com,1999:blog-8669861761761330438.post-8154689431560641398 blogger_orig_url: http://www.michaelsteward.com/2011/10/clearing-confusion-adep-livecycle-day.html --- I’ve done a lot of explaining recently to others regarding the transition to ADEP from LiveCycle by Adobe earlier in the year.  On the face of it there was a big change; the branding is new, the product names changed but underneath it’s still essentially the same albeit with some extra polish and some new features. 
The basic formulae is ADEP = LiveCycle ES2 + CRX (Day)
The move also brings together the old Solution Accelerators (which I’d never previously given a good look) as “Customer Experience Solutions” which are essentially pre-built applications built with ADEP which allow for rapid deployment rather than starting from scratch. 
One of the other bigger changes is in the way ADEP is sold.  It now comes in two flavours: Basic and Standard.  A comparison chart can be found on the Adobe website but essentially the difference is in the number of modules made available (Basic doesn’t include the Process Management module for example).
